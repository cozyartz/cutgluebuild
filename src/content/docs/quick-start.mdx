# Quick Start Guide

Get CutGlueBuild running locally in just a few minutes.

## 5-Minute Setup

### 1. Clone and Install

```bash
git clone https://github.com/cozyartz/cutgluebuild.git
cd cutgluebuild
npm install
```

### 2. Start Development Server

```bash
npm run dev
```

<Badge text="Success!" variant="success" /> Your site is now running at `http://localhost:4321`

### 3. Optional: Add Environment Variables

Create `.env` file for full functionality:

```bash
# Minimal setup - add only what you need
VITE_SITE_URL=http://localhost:4321

# For AI features (optional)
OPENAI_API_KEY=your_openai_key

# For Stripe payments (optional)
VITE_STRIPE_PUBLISHABLE_KEY=your_stripe_key
STRIPE_SECRET_KEY=your_stripe_secret

# Cloudflare credentials (for production deployment)
CLOUDFLARE_ACCOUNT_ID=your_account_id
CLOUDFLARE_API_TOKEN=your_api_token
```

## Common First Tasks

### View the Site
- Homepage: `http://localhost:4321`
- Blog: `http://localhost:4321/blog`
- Templates: `http://localhost:4321/templates`
- Documentation: `http://localhost:4321/docs`

### Make Your First Change
1. Edit `src/pages/index.astro`
2. Change the hero title text
3. Save and see instant updates

### Add a Blog Post
1. Create `src/content/blog/my-first-post.md`
2. Add frontmatter and content
3. View at `http://localhost:4321/blog/my-first-post`

## Development vs Production

### Local Development
- Uses mock database and auth services
- No external dependencies required
- All features work with placeholder data

### Production Deployment
- Requires Cloudflare D1 database setup
- Session-based authentication
- Real Stripe integration for payments

## Setting Up Cloudflare D1 (Optional)

For full database functionality:

```bash
# Install Wrangler CLI
npm install -g wrangler

# Create D1 database
wrangler d1 create cutgluebuild-db

# Update wrangler.toml with your database ID
# Run initial migration
wrangler d1 execute cutgluebuild-db --file migrations/0001_initial_schema.sql

# Populate sample data
wrangler d1 execute cutgluebuild-db --file migrations/0002_populate_templates.sql
```
